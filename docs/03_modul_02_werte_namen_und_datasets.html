Modul 02: Werte, Namen und Datasets

Ziel des Moduls

Die Teilnehmenden lernen, wie Werte und strukturierte Daten in Nova Language modelliert werden und warum dataset fuer den Grundkurs die zentrale Bruecke zu Listen und Dictionaries ist.

Lernziele

  • primitive und strukturierte Werte erkennen
  • Listen und Maps in Nova Language lesen
  • dataset als deklarierte Datenquelle verstehen
  • gute Namen fuer Datensaetze und Felder vergeben

Beispiel

- 02_datasets_und_literale.ns

Fachlicher Kern

Im Python-Kurs spricht man frueh ueber Strings, Integer, Booleans, Listen und Dictionaries. In Nova Language wird dieselbe Denkweise ueber Literale und dataset-Strukturen vermittelt.

Didaktisch wichtig:

  • dataset ist nicht nur Dateiladung.
  • dataset kann Lernenden helfen, strukturierte Eingabedaten klar zu sehen.
  • Eine Liste von Objekten ist fuer viele Uebungen der ideale Einstieg.

Empfohlene Tafel- oder Folienstruktur

  • Text: "Nova Language Grundkurs"
  • Zahl: 10
  • Bool: true
  • Liste: ["intro", "nova", "grundkurs"]
  • Map: {title: "Nova", lessons: 10}

Code verstehen: Was passiert in diesem Beispiel?

Dieses Modul fuehrt den Schritt von einzelnen Werten zu strukturierten Daten ein. Das Beispiel kombiniert ein dataset mit einem kleinen flow, der diese Daten in eine gut lesbare Textform ueberfuehrt.

Der Originalcode

dataset course_info {
  items: [{title: "Nova Language Grundkurs", lessons: 10, active: true, tags: ["intro", "nova", "grundkurs"]}, {title: "Nova Language Uebung", lessons: 3, active: false, tags: ["uebung", "praxis"]}]
}

flow show_course_info {
  py.exec "json.dumps(_, ensure_ascii=False, indent=2)" course_info -> info
  system.log info
}

Zeile fuer Zeile erklaert

  • dataset course_info: Es wird ein benanntes Datenobjekt definiert. Der Name beschreibt, dass es sich um Kursinformationen handelt.
  • items: Das Feld enthaelt eine Liste von Eintraegen. Jeder Eintrag ist eine Map mit klaren Schluesseln wie title, lessons, active und tags.
  • flow show_course_info: Der Flow beschreibt das Verhalten, das auf diesen Daten ausgefuehrt werden soll.
  • py.exec ... course_info -> info: Die Python-Bruecke bekommt das Dataset als Eingabe, serialisiert die Struktur als formatierten JSON-Text und speichert das Ergebnis unter info.
  • system.log info: Der formatierte Text wird sichtbar ausgegeben.

Was passiert zur Laufzeit?

  1. Das Dataset course_info wird als strukturierte Datenquelle bereitgestellt.
  2. Der Flow show_course_info liest dieses Dataset ein.
  3. py.exec sieht die Eingabedaten ueber _ und formt daraus einen lesbaren Text.
  4. Der Alias info uebergibt dieses Zwischenergebnis an system.log.

Denkbruecke zu Python

In Python waere das am ehesten eine Liste von Dictionaries plus json.dumps(...). Nova trennt hier aber sauber zwischen Datenobjekt und Verhalten.

Typisches Missverstaendnis

Ein dataset ist nicht bloss eine Datei und auch nicht einfach nur eine Variable. Es ist ein benannter, deklarierter Datenbaustein im Sprachmodell.

Zentrale Begriffe

  • Literal
  • Struktur
  • Feld
  • Liste
  • Map
  • Dataset

Typische Fehler

  • inkonsistente Feldnamen in Datensaetzen
  • Verwechslung zwischen einem Dataset und einem einzelnen Eintrag
  • unklare Namensgebung wie data1 oder listx

Uebungen

  1. Erweitere das Beispiel um ein drittes Kursobjekt.
  2. Fuege einem Kurseintrag ein weiteres Feld wie level hinzu.
  3. Erklaere, warum items als Liste gut fuer Grundkursbeispiele ist.

Hausaufgabe

Modelliere eine Liste aus drei Buechern oder drei Sensoren als dataset.
