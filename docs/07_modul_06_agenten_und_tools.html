Modul 06: Agenten und Tools

Ziel des Moduls

Die Teilnehmenden lernen, wie Nova Language agentische Bausteine und Werkzeuge als benannte Sprachobjekte beschreibt und in Flows einbindet.

Lernziele

  • tool und agent deklarativ lesen
  • Prompts, Speicher und Werkzeuge als Teile eines Agenten verstehen
  • agentische Schritte im Flow verorten
  • wiederverwendbare Tools von direkter Operation unterscheiden

Beispiel

- 06_agent_und_tool.ns

Fachlicher Kern

Mit diesem Modul erweitert sich der Kurs von klassischen Grundkonzepten auf agentische Programme. Entscheidend ist, dass Agenten nicht mystisch unterrichtet werden, sondern als sauber definierte Sprachobjekte mit Rollen, Speicher und Werkzeugen.

Code verstehen: Was passiert in diesem Beispiel?

Dieses Modul zeigt, dass Nova Language nicht nur Daten und klassische Verarbeitungsschritte beschreiben kann, sondern auch agentisches Verhalten und wiederverwendbare Werkzeuge als Sprachobjekte modelliert.

Der Originalcode

state review_memory {
  namespace: nova_language_reviews
}

tool announce {
  command: system.log {{value0}}
  capability: cpu
}

agent reviewer {
  provider: shell
  model: active
  memory: review_memory
  tools: [announce]
  system_prompt: "Du bist ein einfacher Tutor-Agent fuer den Nova-Language-Grundkurs."
  prompts: {v1: "Fasse den folgenden Input in genau einem Satz zusammen.

{{input}}"}
  prompt_version: v1
}

dataset notes {
  items: [{text: "Nova Language nutzt Flows fuer Verhalten."}, {text: "Datasets beschreiben strukturierte Eingaben."}]
}

flow review_notes {
  reviewer summarize notes -> summary
  announce summary
}

Zeile fuer Zeile erklaert

  • state review_memory: Speicherbereich fuer den Agenten.
  • tool announce: Ein benanntes Werkzeug, das spaeter wiederverwendet werden kann. Hier ist das Werkzeug letztlich eine Log-Ausgabe.
  • agent reviewer: Der Agent wird mitsamt Provider, Modell, Speicher, Werkzeugen und Prompt-Verhalten beschrieben.
  • dataset notes: Die Eingabedaten, die der Agent zusammenfassen soll.
  • reviewer summarize notes -> summary: Der Flow laesst den Agenten auf Basis seiner Definition eine Zusammenfassung erzeugen.
  • announce summary: Das Ergebnis wird nicht direkt via system.log geschrieben, sondern ueber das definierte Tool ausgegeben.

Was passiert zur Laufzeit?

  1. Die Runtime kennt einen Agenten, ein Tool, einen Speicherbereich und ein Dataset.
  2. Beim Start des Flows erhaelt der Agent die Daten aus notes.
  3. Der Agent verarbeitet diese Daten gemaess Prompt und Konfiguration zu summary.
  4. Danach wird das Tool announce mit diesem Ergebnis aufgerufen und fuehrt seine Systemoperation aus.

Denkbruecke zu Python

In Python waere das eine Kombination aus Funktionsaufruf, Tool-Wrapper, Prompt-Konfiguration und Objektzustand. Nova beschreibt diese Rollen deklarativ als erstklassige Sprachbausteine.

Typisches Missverstaendnis

Ein Agent ist hier nicht nur ein Chatfenster mit Textprompt. Er ist ein konfiguriertes Objekt mit Speicher, Werkzeugen und definierter Einbettung in den Datenfluss des Programms.

90-Minuten-Ablauf

Phase 1: Werkzeugdenken

  • Warum ein tool mehr ist als ein einzelner Befehl
  • Wiederverwendung ueber Benennung

Phase 2: Agent lesen

  • Provider, Modell, Speicher und Prompt auseinanderhalten
  • Warum der Agent ein konfiguriertes Objekt ist

Phase 3: Flow analysieren

  • Dataset als Input
  • Agentenschritt als Verarbeitung
  • Tool als Ausgabe

Phase 4: Diskussion

  • Wann ist ein Agent sinnvoll, wann reicht klassische Programmlogik

Zentrale Begriffe

  • Tool
  • Agent
  • Prompt
  • Memory
  • Provider
  • Capability

Typische Fehler

  • Agenten nur als Chatbots missverstehen
  • denken, dass ein Tool immer nur Shell-Syntax sei
  • nicht erkennen, dass Agenten in denselben Datenfluss eingebettet sind wie andere Schritte

Uebungen

  1. Ersetze den Zusammenfassungs-Prompt durch eine Bewertungsfrage.
  2. Fuege ein zweites Tool fuer eine andere Ausgabeform hinzu.
  3. Erweitere das Dataset um einen dritten Merksatz.

Hausaufgabe

Baue ein kleines Beispiel mit einem Agenten, einem Tool und einem Dataset, in dem der Agent einen kurzen Bericht erzeugt.
