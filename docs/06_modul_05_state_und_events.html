Modul 05: State und Events

Ziel des Moduls

Die Teilnehmenden lernen, wie Nova Language Zustand ueber state speichert und Verhalten ueber event reaktiv ausloesen kann.

Lernziele

  • state als persistierbaren Laufzeitzustand verstehen
  • event.emit und Event-Bindungen lesen
  • lokale Aliase und persistierten Zustand unterscheiden
  • reaktive Programme von linearen Skripten unterscheiden

Beispiel

- 05_state_und_events.ns

Fachlicher Kern

Dieses Modul markiert den Uebergang von rein linearen Flows zu Programmen mit Zustand und Reaktivitaet. Lernende sollen erkennen, dass Nova Language nicht nur Berechnungen beschreibt, sondern auch zeitliche und systemische Ausloeser modellieren kann.

Code verstehen: Was passiert in diesem Beispiel?

Dieses Modul fuehrt zwei neue Ebenen ein: persistierbaren Zustand und reaktive Ausloeser. Das Programm berechnet nicht nur einen Wert, sondern speichert ihn und meldet anschliessend ein Ereignis an die Runtime.

Der Originalcode

state lesson_state {
  namespace: nova_language_course
}

dataset metrics {
  items: [{value: 4}, {value: 6}, {value: 10}]
}

flow compute_metrics {
  py.exec "sum(item['value'] for item in _)" metrics -> total
  state.set latest_total total
  event.emit metrics.ready total
}

event schedule_tick {
  on: schedule.tick
  flow: compute_metrics
}

Zeile fuer Zeile erklaert

  • state lesson_state: Ein benannter Zustandsbereich wird definiert. Der Namespace trennt diesen Zustand logisch von anderem Zustand.
  • dataset metrics: Eingabedaten fuer die Berechnung.
  • flow compute_metrics: Verhaltenseinheit fuer die Verarbeitung.
  • py.exec ... metrics -> total: Summe aller value-Felder.
  • state.set latest_total total: Das Zwischenergebnis wird nicht nur lokal benutzt, sondern unter dem Schluessel latest_total im Runtime-Zustand gespeichert.
  • event.emit metrics.ready total: Nach der Berechnung wird ein Ereignis erzeugt, das anderen Teilen des Systems mitteilen kann: Die Metriken sind fertig.
  • event schedule_tick: Diese Deklaration verbindet ein eingehendes Ereignis schedule.tick mit dem Flow compute_metrics.

Was passiert zur Laufzeit?

  1. Ein Tick-Ereignis der Runtime trifft ein.
  2. Dadurch wird compute_metrics gestartet.
  3. Die Daten aus metrics werden summiert.
  4. Das Resultat wird in den Zustand geschrieben.
  5. Anschliessend wird ein neues Ereignis ausgesendet, das die Fertigstellung signalisiert.

Denkbruecke zu Python

In Python waere das eine Mischung aus Variablen, einem Speicherobjekt und einem Callback- oder Event-System. Nova fuehrt diese Aspekte direkt im Sprachmodell zusammen.

Typisches Missverstaendnis

state ist nicht dasselbe wie ein normaler Alias. total ist lokal im Flow. latest_total ist dagegen ein benannter Eintrag im persistierbaren Zustand.

90-Minuten-Ablauf

Phase 1: Zustand einordnen

  • Was ist lokaler Alias
  • Was ist persistierbarer Zustand

Phase 2: Ereignisse lesen

  • Was bedeutet event.emit
  • Wie verbindet ein event einen Ausloeser mit einem Flow

Phase 3: Ablauf analysieren

  • Daten summieren
  • Ergebnis speichern
  • Folgeereignis senden

Phase 4: Transfer

  • Timer, Sensoren oder Statuswechsel als reale Ausloeser diskutieren

Zentrale Begriffe

  • State
  • Namespace
  • Event
  • Emit
  • Reaktivitaet
  • Trigger

Typische Fehler

  • state mit einem normalen Alias verwechseln
  • denken, dass event nur ein Kommentar zur Dokumentation ist
  • reaktive Programme wie rein lineare Skripte lesen

Uebungen

  1. Erweitere das Dataset um einen weiteren Messwert.
  2. Speichere zusaetzlich die Anzahl der Messwerte im Zustand.
  3. Benenne das Folgeereignis so um, dass sein Zweck fachlich klarer wird.

Hausaufgabe

Erstelle ein eigenes Beispiel mit einem state, einem flow und einem event, das auf ein eingehendes Signal reagiert.
