Modul 10: Objektmodell und Komposition

Ziel des Moduls

Die Teilnehmenden lernen, wie Nova Language verschiedene Sprachobjekte zu einem groesseren Gesamtmodell zusammensetzt und wie sich daraus ein eigenes Objektverstaendnis ergibt.

Lernziele

  • System-, State-, Tool-, Agent-, Dataset-, Flow- und Event-Objekte zusammendenken
  • Komposition als zentrales Strukturprinzip erkennen
  • Kontext, Daten und Verhalten unterscheiden
  • das Nova-Objektmodell mit klassischem OOP-Denken vergleichen

Beispiel

- 10_objektmodell_und_komposition.ns

Fachlicher Kern

Dieses Abschlussmodul schliesst den Kreis des Grundkurses. Lernende sehen, dass Nova Language nicht aus isolierten Einzelbefehlen besteht, sondern aus benannten Objekten mit Rollen, Beziehungen und Verhalten. Das ist die Bruecke zum Objektmodell der Sprache.

Code verstehen: Was passiert in diesem Beispiel?

Dieses Abschlussmodul fuehrt viele Sprachobjekte zusammen. Es zeigt, dass Nova-Programme aus mehreren Rollen bestehen, die ueber Komposition verbunden werden: Kontextobjekte, Speicher, Tools, Agenten, Datenquellen, Flows und Events.

Der Originalcode

system course_runtime {
  mode: local
  observability: structured
}

state course_memory {
  namespace: nova_language_object_model
}

tool announce {
  command: system.log {{value0}}
  capability: cpu
  system: course_runtime
}

agent tutor {
  provider: shell
  model: active
  memory: course_memory
  tools: [announce]
  system: course_runtime
  system_prompt: "Du bist ein Tutor fuer das Objektmodell von Nova Language."
  prompts: {v1: "Erklaere kurz, was an diesem Input Daten, Verhalten und Kontext ist.

{{input}}"}
  prompt_version: v1
}

dataset lesson_notes {
  items: [{text: "Ein Dataset ist ein Datenobjekt."}, {text: "Ein Flow beschreibt Verhalten."}, {text: "Ein System beschreibt Kontext."}]
}

flow run_session {
  tutor summarize lesson_notes -> object_view
  state.set last_object_view object_view
  announce object_view
  event.emit session.ready object_view
}

event session_tick {
  on: lesson.tick
  flow: run_session
}

Zeile fuer Zeile erklaert

  • system course_runtime: globaler Kontext fuer die beteiligten Bausteine.
  • state course_memory: persistierbarer Speicher fuer den Agenten und den Ablauf.
  • tool announce: Werkzeug mit Bindung an das definierte System.
  • agent tutor: Komponiert Provider, Modell, Speicher, Tool und Systemkontext zu einem komplexeren Objekt.
  • dataset lesson_notes: Fachlicher Input fuer die Sitzung.
  • flow run_session: Der Agent erzeugt eine Sicht auf den Input, speichert sie, kuendigt sie an und emittiert ein Folgeereignis.
  • event session_tick: Ein externes oder periodisches Ereignis kann die ganze Sitzung ausloesen.

Was passiert zur Laufzeit?

  1. Ein Tick-Ereignis startet den Flow run_session.
  2. Der Agent tutor verarbeitet das Dataset lesson_notes im Kontext seines Systems und Speichers.
  3. Das Ergebnis object_view wird im Zustand gespeichert.
  4. Danach wird es ueber das Tool ausgegeben.
  5. Zum Schluss wird ein neues Ereignis emittiert, das andere Teile der Runtime weiterverarbeiten koennen.

Denkbruecke zu Python

In einer klassischen Sprache waeren hier mehrere Klassen, Konfigurationsobjekte, Service-Wrapper und Event-Handler beteiligt. Nova modelliert diese Beziehungen deklarativ und laesst die Runtime daraus das Gesamtsystem zusammensetzen.

Typisches Missverstaendnis

Komposition bedeutet hier nicht Vererbung oder Klassenhierarchie, sondern das gezielte Zusammensetzen benannter Sprachobjekte mit klaren Rollen und Beziehungen.

90-Minuten-Ablauf

Phase 1: Objekte identifizieren

  • Welche Rolle hat jedes Sprachobjekt
  • Was ist Datenobjekt, was ist Kontextobjekt, was ist Verhalten

Phase 2: Beziehungen lesen

  • Wie Tool und Agent an das System gebunden sind
  • Wie Speicher und Events eingebunden sind

Phase 3: Laufzeitverhalten verfolgen

  • Tick-Ereignis
  • Flow-Ausfuehrung
  • Speichern, Ausgeben, Event emittieren

Phase 4: OOP-Transfer

  • Warum Nova ohne klassische Klassen trotzdem ein starkes Objektdenken erlaubt

Zentrale Begriffe

  • Objektmodell
  • Komposition
  • Kontext
  • Beziehung
  • Rolle
  • Komplexes Sprachobjekt

Typische Fehler

  • nur nach Klassen und Vererbung suchen
  • Objekte ohne ihre Beziehungen lesen
  • die Runtime-Sicht vom Sprachmodell trennen, obwohl beides zusammengehoert

Uebungen

  1. Erweitere das Beispiel um ein zweites Tool.
  2. Fuege ein weiteres Event fuer einen anderen Ausloeser hinzu.
  3. Beschreibe fuer jedes Objekt in einem Satz seine Rolle im Gesamtmodell.

Hausaufgabe

Entwirf ein eigenes Mini-System aus mindestens vier verschiedenen Nova-Objekttypen, die gemeinsam ein Verhalten bilden.
