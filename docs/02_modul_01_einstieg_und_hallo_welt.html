Modul 01: Einstieg und Hallo Welt

Ziel des Moduls

Die Teilnehmenden lernen, was Nova Language ist, wie ein minimales .ns-Programm aussieht und warum ein flow der zentrale Einstiegspunkt fuer Verhalten ist.

Lernziele

  • Nova Language als Teil des Nova-shell-Runtime-Modells einordnen
  • ein minimales .ns-Programm lesen und schreiben
  • den Begriff flow als Verhaltenseinheit verstehen
  • system.log als einfachste sichtbare Aktion nutzen

Beispiel

- 01_hallo_welt.ns

Fachlicher Kern

Im klassischen Python-Grundkurs beginnt man mit print("Hello World"). In Nova Language beginnt der Kurs mit einem flow, der ein sichtbares Verhalten ausfuehrt.

Die entscheidende didaktische Botschaft lautet:

  • Ein Programm in Nova Language ist nicht nur eine Folge von Befehlen.
  • Es beschreibt Verhalten in einer strukturierten, deklarativen Form.
  • Dieses Verhalten wird ueber flow organisiert.

Code verstehen: Was passiert in diesem Beispiel?

Dieses Beispiel ist das kleinstmoegliche sichtbare Nova-Programm. Es zeigt bereits drei Kernideen der Sprache: Verhalten wird in einem flow beschrieben, eine Operation wird ueber ihren fachlichen Kontext angesprochen und das Ergebnis kann ueber einen Alias weitergereicht werden.

Der Originalcode

flow hello_world {
  system.log "Hallo Nova Language" -> message
}

Zeile fuer Zeile erklaert

  • flow hello_world: Hier wird eine benannte Verhaltenseinheit definiert. hello_world ist nicht bloss ein Label, sondern der Name eines ausfuehrbaren Ablaufs.
  • system.log: Die Runtime nimmt aus dem Systembereich die Operation log. Der Code sagt also nicht nur drucke Text aus, sondern nutzt bewusst einen sprachlichen Baustein mit Systemkontext.
  • "Hallo Nova Language": Das ist das Eingabe-Literal fuer diese Operation.
  • -> message: Das Ergebnis des Schritts bekommt den Alias message. Auch wenn das Minimalbeispiel den Alias danach nicht weiter nutzt, zeigt es schon das Datenflussmodell der Sprache.

Was passiert zur Laufzeit?

  1. Die Datei wird geladen und der Flow hello_world wird als Verhaltenseinheit registriert.
  2. Beim Ausfuehren wird die Operation system.log aktiviert.
  3. Der String "Hallo Nova Language" wird als Eingabe uebergeben und ausgegeben.
  4. Das Resultat dieser Operation wird intern unter message verfuegbar gemacht.

Denkbruecke zu Python

In Python waere die sichtbare Wirkung aehnlich zu print("Hallo Nova Language"). Didaktisch ist Nova hier aber groesser gedacht: Nicht nur mache Ausgabe, sondern beschreibe Verhalten, Operation und Ergebnisfluss.

Typisches Missverstaendnis

flow ist hier nicht bloss ein Container und system.log nicht nur ein anderes print. Das Beispiel ist die kleinste Form eines deklarativen Ablaufmodells.

Live-Demo

python nova_shell.py -c "ns.graph .\NovaLanguage\Beispiele\01_hallo_welt.ns"
ns.run .\NovaLanguage\Beispiele\01_hallo_welt.ns

90-Minuten-Ablauf

Phase 1: Einstieg

  • Was ist Nova Language
  • Einordnung in Nova-shell
  • Unterschied zwischen Shell-Kommando und .ns-Programm

Phase 2: Erstes Programm lesen

  • Dateiaufbau erklaeren
  • flow hello_world lesen
  • system.log als sichtbare Wirkung zeigen

Phase 3: Erstes eigenes Programm

  • Text austauschen
  • zweiten flow mit eigener Meldung schreiben
  • Dateinamen und Struktur besprechen

Phase 4: Reflexion

  • Was ist Datenbestandteil
  • Was ist Verhalten
  • Warum ist flow naeher an einer Funktion als an einer Konfigurationszeile

Zentrale Begriffe

  • Nova Language
  • .ns
  • Flow
  • Operation
  • Ausgabe
  • Graph

Typische Fehler

  • flow als blossen Container missverstehen
  • vergessen, dass ein Nova-Programm aus benannten Deklarationen besteht
  • ns.graph und ns.run durcheinanderbringen

Uebungen

  1. Aendere die Begruessung in drei Varianten.
  2. Lege einen zweiten flow mit einer anderen Log-Ausgabe an.
  3. Erklaere in einem Satz, was der Unterschied zwischen .ns und einem direkten Shell-Befehl ist.

Hausaufgabe

Erstelle eine Datei mein_erstes_programm.ns mit zwei Flows, die unterschiedliche Log-Meldungen ausgeben.

Transferfrage

Warum ist ein flow in Nova Language kein exakter Ersatz fuer print, sondern eher Teil einer groesseren Programmbeschreibung
