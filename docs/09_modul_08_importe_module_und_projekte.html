Modul 08: Importe, Module und Projekte

Ziel des Moduls

Die Teilnehmenden lernen, wie .ns-Programme ueber mehrere Dateien organisiert werden und warum Modularisierung auch in Nova Language fuer Lesbarkeit und Wartbarkeit entscheidend ist.

Lernziele

  • import lesen und einsetzen
  • gemeinsame Bausteine auslagern
  • Dateien sinnvoll benennen und strukturieren
  • ein kleines Mehrdatei-Projekt ueberblicken

Beispiele

  • 08_import_shared.ns
  • 08_import_main.ns

Fachlicher Kern

Sprachunterricht auf Grundkursniveau sollte spaetestens an dieser Stelle zeigen, dass gutes Programmieren immer auch gute Struktur bedeutet. Nova Language hat dafuer den gleichen Anspruch wie andere Sprachen:

  • nicht alles in eine Datei
  • Bausteine gezielt auslagern
  • Wiederverwendung und Lesbarkeit verbessern

Code verstehen: Was passiert in diesem Beispiel?

Dieses Modul zeigt Modularisierung in Nova Language. Ein Teil der Bausteine liegt in einer gemeinsamen Datei, die Hauptdatei importiert diese Deklarationen und verwendet sie im eigenen Flow.

Die gemeinsame Datei

dataset shared_notes {
  items: [{text: "Importe machen Nova-Programme modular."}]
}

agent helper {
  provider: shell
  model: active
  system_prompt: "Du bist ein einfacher Hilfsagent fuer Unterrichtsbeispiele."
  prompts: {v1: "Fasse den Input kurz zusammen.

{{input}}"}
  prompt_version: v1
}

Die Hauptdatei

import "08_import_shared.ns"

system course_imports {
  namespace: nova_language_imports
}

flow imported_demo {
  helper summarize shared_notes -> summary
  system.log summary
}

Zeile fuer Zeile erklaert

  • import "08_import_shared.ns": Die Datei bindet externe Deklarationen ein.
  • shared_notes und helper werden dadurch in der Hauptdatei nutzbar.
  • system course_imports: Definiert zusaetzlich lokalen Kontext fuer dieses Import-Beispiel.
  • helper summarize shared_notes -> summary: Der importierte Agent verarbeitet das ebenfalls importierte Dataset.
  • system.log summary: Das Ergebnis wird in der Hauptdatei ausgegeben.

Was passiert zur Laufzeit?

  1. Die Runtime laedt zuerst die Hauptdatei.
  2. Beim Import wird die Shared-Datei aufgeloest und ihre Bausteine werden in das Gesamtmodell eingebunden.
  3. Danach kann der Flow so arbeiten, als waeren Agent und Dataset lokal vorhanden.
  4. Der Agent erzeugt eine Zusammenfassung, die anschliessend geloggt wird.

Denkbruecke zu Python

Das entspricht grundsaetzlich dem Gedanken hinter import in Python: Definitionen werden in andere Dateien ausgelagert. Nova uebernimmt also auch fuer Mehrdatei-Projekte die Idee sauberer Modularisierung.

Typisches Missverstaendnis

Importe sind nicht nur Bequemlichkeit. Sie sind die Grundlage dafuer, Nova-Programme in echte Projektstrukturen mit Wiederverwendung und Trennung von Verantwortung zu ueberfuehren.

Live-Demo

@'
from pathlib import Path
from nova.runtime.runtime import NovaRuntime
path = Path(r".\NovaLanguage\Beispiele\08_import_main.ns").resolve()
source = path.read_text(encoding="utf-8")
with NovaRuntime() as runtime:
    runtime.load(source, source_name=str(path), base_path=path.parent)
    print("Mehrdatei-Programm erfolgreich geladen.")
'@ | python -

90-Minuten-Ablauf

Phase 1: Problemstellung

  • Wann wird eine Datei zu gross
  • Warum Wiederverwendung wichtig ist

Phase 2: Importsyntax

  • importierte Datei
  • gemeinsame Deklarationen
  • Hauptdatei mit Flow

Phase 3: Projektstruktur

  • Shared-Datei
  • Main-Datei
  • Benennungskonventionen

Phase 4: Praxis

  • Agent und Dataset in Shared-Datei
  • Flow in Main-Datei

Zentrale Begriffe

  • Import
  • Modul
  • Projektstruktur
  • Wiederverwendung
  • Hauptdatei

Typische Fehler

  • zu grosse Sammeldateien
  • unklare Dateinamen
  • Zirkularitaet oder unnoetige Abhaengigkeiten

Uebungen

  1. Lagere einen weiteren Baustein in die Shared-Datei aus.
  2. Erstelle eine zweite Main-Datei mit anderer Nutzung derselben Bausteine.
  3. Benenne die Dateien fuer ein Schulungsprojekt um.

Hausaufgabe

Baue ein Mini-Projekt mit mindestens zwei .ns-Dateien, einer Shared-Datei und einer Hauptdatei.
