Modul 07: Systeme, Packages und Services

Ziel des Moduls

Die Teilnehmenden lernen, dass Nova Language auch Plattform- und Runtime-Strukturen beschreibt und damit ueber das hinausgeht, was in vielen Einfuehrungssprachen ueblich ist.

Lernziele

  • system, package und service deklarativ einordnen
  • den Zusammenhang zwischen Programmlogik und Laufzeitkontext verstehen
  • Statusabfragen in Flows lesen
  • einfache Plattformmodelle erklaeren

Beispiel

- 07_system_service_package.ns

Fachlicher Kern

In Nova Language gehoeren technische Kontexte zum Sprachmodell. Dieses Modul sollte Dozierenden helfen, die Bruecke zwischen Programmiersprache und Runtime-Denken zu schlagen:

  • system beschreibt Kontext
  • package beschreibt Artefakte
  • service beschreibt deployte oder laufende Einheiten

Didaktischer Hinweis

Dies ist kein Einstiegsmodul fuer absolute Programmierneulinge. Es ist ein spaeter Grundkursblock, in dem gezeigt wird, dass Nova Language eine Programmiersprache fuer Daten, Verhalten und Plattformkontext zugleich ist.

Code verstehen: Was passiert in diesem Beispiel?

Hier wird Nova Language als Sprache fuer Programmlogik und Laufzeitkontext sichtbar. Das Beispiel definiert System-, Paket- und Serviceobjekte und liest danach ihren Status aus.

Der Originalcode

system school_core {
  tenant: academy
  namespace: training
  observability: structured
}

package teaching_sdk {
  version: 1.0.0
  source: ./dist/teaching-sdk.tar
  auto_install: true
}

service training_api {
  package: teaching_sdk
  image: nova:training-api
  replicas: 2
  strategy: blue_green
  auto_deploy: true
}

flow inspect_platform {
  package.status teaching_sdk -> pkg
  service.status training_api -> svc
  state.set last_service_status svc
}

Zeile fuer Zeile erklaert

  • system school_core: Beschreibt globalen Plattformkontext wie Tenant, Namespace und Beobachtbarkeit.
  • package teaching_sdk: Modelliert ein Artefakt, das installiert oder genutzt werden kann.
  • service training_api: Beschreibt eine laufende beziehungsweise deploybare Einheit, die auf dem Paket basiert.
  • package.status teaching_sdk -> pkg: Der Flow fragt den aktuellen Paketstatus ab.
  • service.status training_api -> svc: Danach wird der Status des Services gelesen.
  • state.set last_service_status svc: Der zuletzt gelesene Servicestatus wird fuer spaetere Nutzung gespeichert.

Was passiert zur Laufzeit?

  1. Die Runtime laedt die Plattformobjekte und ihre Beziehungen.
  2. Der Flow fragt den Status des Pakets und des Services ab.
  3. Die Resultate werden unter Aliasen verfuegbar.
  4. Der Servicestatus wird in den Zustand uebernommen.

Denkbruecke zu Python

In einer klassischen Python-Anwendung waeren solche Informationen oft auf mehrere YAML-, JSON- oder Deployment-Dateien verteilt. Nova zieht dieses Runtime-Wissen in das Sprachmodell selbst.

Typisches Missverstaendnis

system, package und service sind keine Kommentare zur Umgebung. Sie sind echte Sprachobjekte, die vom Laufzeitmodell verstanden werden.

Live-Demo

python nova_shell.py -c "ns.graph .\NovaLanguage\Beispiele\07_system_service_package.ns"
ns.run .\NovaLanguage\Beispiele\07_system_service_package.ns

90-Minuten-Ablauf

Phase 1: Begriffsklaerung

  • Was ist ein Systemkontext
  • Was ist ein Paket
  • Was ist ein Service

Phase 2: Beispielgraph lesen

  • Deklarationen einzeln identifizieren
  • Beziehungen zwischen den Objekten markieren

Phase 3: Flow mit Statusabfragen

  • package.status
  • service.status
  • state.set

Phase 4: Diskussion

  • Warum ist das in der Sprache und nicht nur in externer YAML oder Shell-Skripten

Zentrale Begriffe

  • System
  • Package
  • Service
  • Runtime-Kontext
  • Deployment-Sicht

Typische Fehler

  • system als blosse Kommentarhülle behandeln
  • package und service durcheinanderwerfen
  • Plattformobjekte zu frueh vor Grundlagenthemen einfuehren

Uebungen

  1. Benenne system, package und service neu fuer ein Schulungsprojekt.
  2. Fuege einen zweiten Service hinzu.
  3. Diskutiere, welche Informationen in system und welche in service gehoeren.

Hausaufgabe

Entwirf ein kleines Plattformmodell fuer eine Trainings- oder Monitoring-Umgebung mit einem system, einem package und einem service.
