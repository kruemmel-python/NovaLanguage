Modul 09: Python-Bruecke und Transformationen

Ziel des Moduls

Die Teilnehmenden verstehen, warum py.exec in Nova Language ein wichtiges Werkzeug ist, aber nicht das Zentrum der Sprache darstellt.

Lernziele

  • py.exec gezielt lesen und einsetzen
  • Nova Language und eingebettete Python-Transformation unterscheiden
  • kleine Transformationen sauber formulieren
  • Grenzen und Risiken ueberladener py.exec-Bloecke erkennen

Beispiel

- 09_python_bruecke.ns

Fachlicher Kern

Die Python-Bruecke ist fuer den Grundkurs didaktisch sehr wertvoll:

  • Lernende mit Python-Vorkenntnissen finden sofort einen Anknuepfungspunkt.
  • Gleichzeitig lernen sie, dass Nova Language selbst das uebergeordnete Strukturmodell liefert.

Die Leitregel fuer Dozierende lautet:

  • so viel Nova wie moeglich
  • so viel Python wie noetig

Code verstehen: Was passiert in diesem Beispiel?

Dieses Modul zeigt die Python-Bruecke als pragmatisches Werkzeug. Nova bleibt die aeussere Struktur, waehrend Python kleine Transformationen uebernimmt.

Der Originalcode

dataset records {
  items: [{name: "Anna", value: 5}, {name: "Boris", value: 3}, {name: "Anna", value: 7}]
}

flow normalize_records {
  py.exec "{item['name'].lower(): item['value'] for item in _}" records -> last_value_by_name
  py.exec "'Eintraege: {}'.format(len(_))" records -> count_text
  system.log last_value_by_name
  system.log count_text
}

Zeile fuer Zeile erklaert

  • dataset records: Liste von Datensaetzen mit Namen und Werten.
  • py.exec ... records -> last_value_by_name: Aus der Liste entsteht eine Map mit kleingeschriebenen Namen als Schluessel. Weil Anna zweimal vorkommt, bleibt der spaetere Wert bestehen.
  • py.exec ... records -> count_text: Gleichzeitig wird ein kurzer Bericht ueber die Anzahl der Eintraege gebildet.
  • system.log ...: Beide Ergebnisse werden ausgegeben.

Was passiert zur Laufzeit?

  1. Die Runtime uebergibt das Dataset an zwei kleine Python-Transformationen.
  2. Die erste berechnet eine normalisierte Nachschlagstruktur.
  3. Die zweite erzeugt einen lesbaren Textbericht.
  4. Nova uebernimmt anschliessend wieder die Weitergabe und Ausgabe dieser Ergebnisse.

Denkbruecke zu Python

Didaktisch ist das der Punkt, an dem Python-Vorkenntnisse helfen. Trotzdem bleibt wichtig: Python loest hier nur lokale Transformationsaufgaben. Das Gesamtmodell aus Datenquelle, Schritten, Aliasen und Ausgabe bleibt Nova.

Typisches Missverstaendnis

Wer hier nur auf die Python-Snippets schaut, verpasst den eigentlichen Punkt des Moduls. Nova ist nicht der Container fuer Python, sondern die Sprache, die den uebergeordneten Ablauf modelliert.

Live-Demo

python nova_shell.py -c "ns.graph .\NovaLanguage\Beispiele\09_python_bruecke.ns"
ns.run .\NovaLanguage\Beispiele\09_python_bruecke.ns

90-Minuten-Ablauf

Phase 1: Einordnung

  • Warum gibt es py.exec
  • Welche Aufgaben sind dafuer geeignet

Phase 2: Beispiel lesen

  • Eingabedaten
  • Transformation zu einer Map
  • textuelle Zusammenfassung

Phase 3: Gute und schlechte Nutzung

  • gut: kleine, klare Transformation
  • schlecht: komplette Programmlogik in einem einzigen Python-Block

Phase 4: Refactoring-Übung

  • zu lange py.exec-Ausdruecke kuerzen
  • Zwischenschritte mit Aliasen benennen

Zentrale Begriffe

  • Python-Bruecke
  • Transformation
  • Interoperabilitaet
  • Pragmatismus
  • Lesbarkeit

Typische Fehler

  • Nova Language nur als Container fuer Python behandeln
  • unlesbare Monolithen in py.exec schreiben
  • nicht mehr unterscheiden, welche Struktur im Flow gerade vorliegt

Uebungen

  1. Baue eine zweite Transformation ein, die den Mittelwert berechnet.
  2. Teile einen langen py.exec-Schritt in zwei klar benannte Schritte auf.
  3. Entscheide bei drei Beispielen, was besser in Nova und was besser in Python aufgehoben ist.

Hausaufgabe

Nimm ein kleines dataset und erstelle zwei alternative Loesungen: einmal mit einer knappen Python-Bruecke und einmal mit maximaler Klarheit im Datenfluss.
