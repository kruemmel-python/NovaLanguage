Modul 03: Flows, Aliase und Datenfluss

Ziel des Moduls

Die Teilnehmenden verstehen, wie Daten in einem flow von Schritt zu Schritt weitergereicht werden und welche Rolle Aliase fuer Lesbarkeit und Denkstruktur spielen.

Lernziele

  • einen mehrstufigen flow lesen
  • den Pfeil -> als Aliasbildung verstehen
  • Zwischenergebnisse sinnvoll benennen
  • Datenfluss statt nur Befehlsabfolge erkennen

Beispiel

- 03_alias_und_datenfluss.ns

Fachlicher Kern

Dies ist das erste Modul, in dem sich Nova Language klar von einem klassischen Skriptkurs unterscheidet. Lernende sollen verstehen:

  • Ein Flow ist eine Folge semantischer Verarbeitungsschritte.
  • Ein Alias wie -> total benennt das Ergebnis eines Schritts.
  • Gute Aliasnamen sind fuer Lesbarkeit und Wartung entscheidend.

Didaktischer Vergleich mit Python

Python:

total = sum(item["value"] for item in numbers)
summary = f"Gesamtsumme: {total}"

Nova Language:

py.exec "sum(item['value'] for item in _)" numbers -> total
py.exec '"Gesamtsumme: {}".format(_)' total -> summary

Code verstehen: Was passiert in diesem Beispiel?

Hier wird Nova Language erstmals als echte Datenfluss-Sprache sichtbar. Das Beispiel nimmt strukturierte Eingabedaten, berechnet daraus eine Summe, formt daraus einen Berichtstext und gibt diesen anschliessend aus.

Der Originalcode

dataset numbers {
  items: [{value: 3}, {value: 5}, {value: 8}]
}

flow sum_numbers {
  py.exec "sum(item['value'] for item in _)" numbers -> total
  py.exec "'Gesamtsumme: {}'.format(_)" total -> summary
  system.log summary
}

Zeile fuer Zeile erklaert

  • dataset numbers: Eingabedaten als Liste von Objekten mit dem Feld value.
  • py.exec ... numbers -> total: Die Liste wird an Python uebergeben. Dort werden alle Werte aufsummiert. Das Ergebnis bekommt den Alias total.
  • py.exec ... total -> summary: Der vorherige Alias wird zur neuen Eingabe. Aus der Zahl entsteht ein Text.
  • system.log summary: Der Berichtstext wird ausgegeben.

Was passiert zur Laufzeit?

  1. Die Runtime kennt das Dataset numbers und den Flow sum_numbers.
  2. Der erste Schritt zieht die Liste aus dem Dataset und berechnet die Summe.
  3. Der Alias total speichert das Zwischenergebnis fuer den naechsten Schritt.
  4. Der zweite Schritt erzeugt aus der nackten Zahl eine sprechende Ausgabe.
  5. Der dritte Schritt macht dieses Endergebnis sichtbar.

Denkbruecke zu Python

In Python wuerde man typischerweise eine Variable total und danach eine Variable summary schreiben. Nova zeigt dieselbe Logik als benannten Strom von Zwischenergebnissen.

Typisches Missverstaendnis

Der Pfeil -> ist kein Dekorationszeichen. Er ist der Kern des Beispiels, weil er das Resultat eines Schritts explizit benennt und damit den Datenfluss lesbar macht.

90-Minuten-Ablauf

Phase 1: Von Daten zu Verhalten

  • dataset als Eingang
  • flow als Verarbeitung

Phase 2: Aliasbildung erklaeren

  • Was bedeutet -> total
  • Warum total besser ist als x1

Phase 3: Datenfluss visualisieren

  • Eingang numbers
  • Transformation zu total
  • Transformation zu summary
  • Ausgabe

Phase 4: Eigene Varianten

  • Durchschnitt statt Summe
  • Maximum statt Summe
  • Textuelle Ergebnisdarstellung anpassen

Zentrale Begriffe

  • Alias
  • Datenfluss
  • Transformation
  • Zwischenergebnis
  • Lesbarkeit

Typische Fehler

  • Aliasnamen ohne Aussagekraft
  • Eingabe _ und benannte Aliase verwechseln
  • zu viel Python-Logik in einen einzigen Schritt packen

Uebungen

  1. Baue aus dem Beispiel eine Durchschnittsberechnung.
  2. Fuege einen weiteren Schritt hinzu, der die Anzahl der Werte berechnet.
  3. Benenne die Aliase so um, dass ein Dritter den Flow sofort versteht.

Hausaufgabe

Schreibe einen flow, der aus einem dataset mit Punkten die Gesamtsumme und einen kurzen Bericht erzeugt.
